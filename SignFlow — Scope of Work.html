1. Project Overview

SignFlow is a custom SaaS platform to manage real estate sign installation and removal operations for Sign Up Sign Services Inc in Tucson, Arizona. The platform replaces a third-party system (UpSignDown) and gives the client full ownership of their data, workflows, and billing operations.

Sign Up Sign Services installs, maintains, and removes real estate yard signs for real estate offices and independent agents throughout the Tucson metro area and surrounding communities including Green Valley. The business manages a team of field installers, a roster of real estate offices and hundreds of agents, and a high daily volume of installation, removal, service call, and delivery orders.

The current third-party platform (UpSignDown) costs approximately $100/month plus 1% on all credit card transactions plus $1 per order. SignFlow will replace it entirely, giving the client full ownership of all data and operations.

2. Background & Business Motivation

Sign Up Sign Services has outgrown its current third-party platform and requires a purpose-built solution that provides full data ownership, modern workflows, and the operational efficiency needed to scale. Key drivers for building SignFlow include:

  • Eliminating ongoing third-party licensing fees and per-transaction charges
  • Full ownership and control of all business data
  • Office-level post type menus and bulk pricing management
  • A streamlined installer mobile experience with route-ordered stops and pull lists
  • Integrated photo confirmation workflow
  • Built-in invoicing, batch billing, and payment processing
  • The ability to schedule removals directly from completed installs
  • A modern, efficient admin experience built around actual daily workflows

3.2 Agent Portal (Web — Desktop & Mobile browser)

FeatureDescription
DashboardActive orders and order history. Place order CTA at top. Stats: active orders, sign panels, orders this month, or balance due for independent agents.
Order form6-section scrollable form, all sections open simultaneously. Order type (install/remove/service call/delivery), property address with map pin, post type selection filtered to office menu and prices, sign panel, accessories, photo confirmation pre-checked. Service calls include required issue description and optional replacement sign selection. Install notes carry over to removal orders automatically.

Charge breakdown (critical data architecture requirement): Every order must store all charge components as separate fields in the database even though the agent sees only one total. Required fields per order: (1) Post install charge — taxable, (2) Zone fee — not taxable, determined automatically by the zone boundary the property address falls in, (3) Photo fee — not taxable, (4) Accessories total — not taxable, (5) Grand total shown to agent — sum of all above. The agent-facing UI displays one combined total only. The admin and reporting layer has access to all individual components for sales tax reporting and auditing purposes.

4. Out of Scope

The following are explicitly excluded from this engagement. Any additions require a written change order.
  • Native iOS or Android app (installer app is web-based mobile browser)
  • QuickBooks or accounting software integration
  • SMS/text messaging to agents or installers
  • Marketing tools (e-flyers, CALLcapture, QR codes)
  • Audio readout of order notes
  • Multi-region / multi-city support (Tucson only at launch)
  • Agent-to-agent sign transfers or marketplace features
  • Automated route optimization engine (manual zone assignment at launch)
  • Custom reporting or analytics beyond what is specified

5. Technical Requirements

Platform

  • Web application — admin and agent portals accessible via desktop browser
  • Installer app — responsive web app optimized for mobile browser (iOS Safari + Android Chrome)
  • No native app builds required at launch

Hosting & Infrastructure

  • Cloud-hosted (AWS, GCP, or Azure — vendor recommendation)
  • SSL/HTTPS on all endpoints
  • Automated daily database backups
  • 99.5% uptime SLA minimum

Security

  • Role-based access control: Admin, Agent, Installer
  • PCI-compliant card handling via Stripe — no raw card data stored
  • Secure session management with auto-logout

Admin Access

  • Client retains full admin access to all data at all times
  • Admin must be able to export all data to Excel at any time
  • No vendor lock-in on data — client owns all data stored in the platform

Browser Support

  • Admin portal: Chrome, Safari, Firefox, Edge (latest 2 versions)
  • Agent portal: Same as admin, plus mobile Safari and mobile Chrome
  • Installer app: Mobile Safari (iOS) and Chrome (Android)

7. Ownership & Intellectual Property

  • All code, designs, and deliverables produced under this agreement are owned exclusively by Sign Up Sign Services Inc upon full payment
  • Vendor retains no license to use, resell, or repurpose any deliverable without written consent
  • All data entered into the platform by the client, agents, or installers is owned by Sign Up Sign Services Inc
  • Client may request a full data export at any time at no charge
  • Source code must be delivered to client at the conclusion of each phase via a private GitHub repository to which the client has full access
  • Vendor must not use any proprietary business logic, pricing structures, or workflow designs from this project in competing products

8. Assumptions & Dependencies

Client responsibilities

  • Client will provide all content, logos, and branding assets within 5 business days of request
  • Client will provide timely feedback (within 3 business days) at each review milestone
  • Client will set up Stripe account and provide API keys
  • Client will manage data entry of any additional offices, agents, and sign panels not migrated from the existing system

Vendor responsibilities

  • Vendor will set up Google Maps API account on behalf of the client and configure all required API keys
  • Vendor will manage the transition from the existing platform (UpSignDown). Client will request a full data export from UpSignDown. If a complete data export is not available, vendor may be needed to assist in re-creating data in the new system
  • Vendor will treat the provided prototype as the primary design specification
  • Vendor will flag any technical constraints that would prevent implementing a prototyped feature before beginning that feature
  • Vendor will provide weekly status updates including hours burned, hours remaining, and blockers
  • Vendor will maintain a private GitHub repository with current source code accessible to the client at all times
  • Vendor will not begin work on a new phase until the prior phase has been accepted in writing by the client
The interactive prototype represents the agreed-upon design for all screens. Any deviation from the prototype requires explicit written approval from the client prior to implementation.
